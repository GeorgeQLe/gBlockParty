- Phase 1 / Step 1.4: Rewrote `packages/gblock-schema/src/index.ts` as `z.discriminatedUnion("type", […])` over 8 variant schemas built from a shared `baseFields` spread. `episode` video-or-audio requirement is enforced via union-level `.superRefine()` (not `.refine()` on the variant) because Zod 3.25's `discriminatedUnion` refuses `ZodEffects` members — the earlier plan's assumption that `.refine()` on a variant worked turned out to be false in this Zod line; `.superRefine()` at the union level preserves per-variant strictness and keeps the union shape intact. `gBlockTypeSchema` expanded to the full 8-type enum (`clip`, `stream` added). Shared optional surface fields (`heroImage`, `videoUrl`, `featured`, `seriesSlug`) now live on every variant via the base spread. Also fixed a pre-existing Step 1.2 typecheck bug in `packages/gblock-schema/src/__tests__/schema.test.ts` — factory helpers typed their `overrides` as `Record<string, unknown>`, which made destructures like `const { videoUrl, ...rest } = stream()` fail TS2339; switched to generic `<T extends Record<string, unknown>>` so spread preserves keys in the return type. `pnpm --filter @gblockparty/gblock-schema test` exits 0 (14/14); `pnpm --filter @gblockparty/gblock-schema typecheck` clean. `pnpm -w test` still non-zero — only the Step 1.3 loader suite is red (unresolved `../loader` / `../collections`); schema regressions: none. `pnpm -w -r typecheck` — schema package clean; `apps/web` still red on the same loader imports (carried over from Step 1.3, Step 1.6 scope).
