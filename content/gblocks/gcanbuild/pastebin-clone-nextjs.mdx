## What we're building

A fully functional pastebin application where users can sign in, create pastes, view them on a personal dashboard, and share them via URL. This is a beginner-friendly, step-by-step walkthrough covering the full stack — from auth to database to deployment.

<Callout tone="info">
This tutorial builds on a starting-point repo that has the base Next.js + Shadcn/ui project scaffolded. Grab it from [the starting point repo](https://github.com/GeorgeQLe/walkthrough-starting-point) or follow the [companion setup video](https://youtu.be/5vdooX7g7AE) first.
</Callout>

## Stack

- **Next.js 15** — App Router for server and client rendering
- **Better Auth** — Social OAuth with GitHub and Google providers
- **Neon** — Serverless Postgres database
- **Drizzle ORM** — Type-safe schema, queries, and migrations
- **tRPC** — End-to-end typesafe API layer
- **Tanstack Query** — Data fetching, caching, and prefetching on the client
- **Shadcn/ui** — Prebuilt UI components for forms and tables
- **Vercel** — Production deployment

## Step 1 — Authentication with Better Auth

Set up Better Auth with GitHub and Google OAuth providers. This gives users a login page with social sign-in buttons, session management, and logout functionality. The auth layer uses Drizzle under the hood so all session and account data lives in your own Postgres database — no third-party session stores.

## Step 2 — Drizzle schema and Neon database

Define the Drizzle ORM schema for the `pastes` table, connecting it to the user model that Better Auth manages. Run migrations against a Neon serverless Postgres instance. Drizzle's type-safe schema means your TypeScript types stay in sync with the database automatically.

## Step 3 — tRPC API procedures

Wire up tRPC with Tanstack Query to create typesafe API procedures for creating and fetching pastes. Mutations handle form submissions while queries power the dashboard. Prefetching on the server means the dashboard loads with data already hydrated — no loading spinners on first paint.

## Step 4 — Forms and dashboard UI

Build the paste creation form using React Hook Form with Shadcn/ui input components. The dashboard uses a Shadcn data table to display all of the current user's pastes with columns for title, creation date, and actions. Authorized access ensures users only see their own pastes.

<Callout tone="info">
React Hook Form pairs well with Shadcn's form primitives — you get validation, error messages, and accessible labels with minimal boilerplate.
</Callout>

## Step 5 — Deploy to Vercel

Push the project to GitHub and deploy to Vercel. Connect your Neon database via environment variables, configure the OAuth callback URLs for production, and verify the full flow end-to-end: sign in, create a paste, view the dashboard, and share a paste URL.
